``ConsensusCalls`` and ``ConsensusConfidence``
----------------------------------------------

.. warning:: 

    The ``ConsensusCalls`` and ``ConsensusConfidence``
    datasets are deprecated.

Post-mapping, the cmp.h5 may be consumed by an algorithm that produces
a consensus for a given reference. The consensus sequence and
corresponding P-values will be recorded in the .cmp.h5 file.

A ``Consensus`` group is defined under the reference group. Consensus
group contains two datasets, ConsensusCalls and ConsensusConfidence.
Consensus groups have several attributes, namely:

Algorithm Source - the name and version of the algorithm being used to
perform consensus calling

Sequence Sample Ploidy - whether the sample is haploid or diploid

Date Generated - time stamp for consensus calling results

ConsensusCalls is a single dimensional byte array that represents the
consensus call at each position with one byte. The compression
strategy is essentially the same as that used in the readBas-refBase
alignment, except that the first four bits are set by allele 1 and the
last four bits are set by the allele 2. If the sample was haploid,
then the first four bits would be untouched and the last four bits
would be set by the only allele present.

Byte Setting:

Diploid

Allele 1 Allele 2

------------------ ------------------

0 0 0 0 0 0 0 0

T G C A T G C A

Haploid

Allele

------------------

0 0 0 0 0 0 0 0

T G C A T G C A

ConsensusConfidence is a single dimensional float
array that stores the p-value associated with the
compressed call at each position. There is a 1:1
row correspondence between this dataset and the
ConsensusCalls dataset.

The explicit definition of HDF5 type for these two
datasets is shown below::

    DATASET "/refSeq/Consensus/ConsensusCalls" {

    DATATYPE H5T_STD_U8LE

    DATASPACE SIMPLE { ( 100 ) / ( H5S_UNLIMITED ) }

    }

    DATASET "/refSeq/Consensus/ConsensusConfidence" {

    DATATYPE H5T_IEEE_F32LE

    DATASPACE SIMPLE { ( 100 ) / ( H5S_UNLIMITED ) }

    }



Specification for the SAF used for Strobe Analysis 
--------------------------------------------------

.. warning:: 

    Strobe analyis is deprecated.

In addition to all datasets specified for the
standard SAF, the following additional datasets
are required for Strobe SAF files.

1. Within the info group ``MovieInfo``

  - ``WhenStarted``: The relative time in seconds from hot start until
    movie acquisition. This field is determined via the primary
    "WhenStarted" field located in ``ScanData/AcqParams/`` from
    bas.h5/pls.h5 by comparing ``WhenStarted`` to the time at which
    the chip was put on the stage.

    Data type and data space definition::

    DATASET "/MovieInfo/MovieStartTime" {

    DATATYPE H5T_IEEE_F32LE

    DATASPACE SIMPLE { ( * ) / ( H5S_UNLIMITED ) }

    }

2. Within the info group ``AlnInfo`` containing information about each
   alignment stored in the file

  - ``StartTimeOffset``: a float32 dataset containing the StartTime
    for the first aligned base.

    Data type and data space definition::

    DATASET "/AlnInfo/StartTimeOffset" {

    DATATYPE H5T_IEEE_F32LE

    DATASPACE SIMPLE { ( * ) / ( H5S_UNLIMITED ) }

    }
